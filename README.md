# Tiny Self-Hosted Analytics (PHP + SQLite)

Framework-free analytics for simple hosting setups.

## Features

- Tiny tracker (`tracker.js`) with `navigator.sendBeacon()` first
- Tracks pageviews, tagged clicks, and custom events
- PHP ingest endpoint (`collect.php`) with PDO prepared statements
- SQLite storage with indexed reporting queries
- Password-protected admin dashboard
- One-time browser setup via `setup.php`
- Basic bot estimation (`human`, `bot`, `unknown`)
- Origin/site allowlist guard on ingest

## Requirements

- PHP 8.0+ (with `pdo_sqlite` extension)
- Writable directory for SQLite file (default: `data/`)
- Web server that can serve PHP files

## File Structure

- `setup.php` one-time installer
- `collect.php` event ingest endpoint
- `tracker.js` analytics tracker
- `schema.sql` SQLite schema + indexes
- `config.example.php` config template
- `config.php` generated by installer
- `lib/bootstrap.php` config/db/helpers
- `lib/auth.php` admin auth helpers
- `lib/bot.php` bot classifier
- `admin/login.php` admin login
- `admin/index.php` dashboard
- `admin/logout.php` logout
- `data/.htaccess` protects DB dir on Apache

## Quick Start

1. Upload files to your analytics host (for example `data.example.com`).
2. Open `https://data.example.com/setup.php`.
3. Fill in:
   - SQLite DB path (`data/analytics.sqlite` is fine)
   - Primary site/domain (for example `example.com`)
   - Allowed origins (for example `https://example.com`)
   - Admin username/password
4. Submit setup form.
5. Open `https://data.example.com/admin/login.php`.

## DNS / Subdomain Setup

Use a dedicated analytics subdomain (for example `data.example.com`).

- If app runs on your own server: create `A`/`AAAA` record.
- If provider/proxy requires aliasing: create `CNAME` record.

Example:

- `data.example.com -> A 203.0.113.10`
- or `data.example.com -> CNAME analytics-host.provider.net`

## Add Tracker to Your Site

Add to pages on your main site (for example `example.com`):

```html
<script>
  window.TINY_ANALYTICS_ENDPOINT = 'https://data.example.com/collect.php';
  window.TINY_ANALYTICS_SITE = 'example.com';
</script>
<script async src="https://data.example.com/tracker.js"></script>
```

### Tagged Click Tracking

Only elements with `data-analytics-click` are tracked as click events:

```html
<a href="/pricing" data-analytics-click="pricing_cta">See pricing</a>
<button data-analytics-click="signup_button" data-analytics-value="hero">Sign up</button>
```

### Custom Events

```html
<script>
  window.TinyAnalytics.track('video_play', 'homepage_intro');
</script>
```

## Ingest API Contract

`POST /collect.php` JSON body:

- Required:
  - `event_type`: `pageview | click | custom`
  - `page_url`
- Optional:
  - `referrer`
  - `event_name`
  - `event_value`
  - `ts`
  - `site`

Responses:

- `204` success
- `400` invalid payload
- `403` disallowed origin/site
- `405` method not allowed
- `500` server/database error
- `503` app not installed

## Bot Estimation

Simple heuristic:

- `bot` if user-agent matches common crawler/headless markers
- `human` if browser-like
- `unknown` otherwise

## Dashboard Metrics

The admin dashboard shows:

- Events today
- Events last 7 days
- Human vs bot vs unknown (7 days)
- Event counts by type (7 days)
- Top pages (7 days)
- Top referrers (7 days)
- Daily counts (7 days)

## Security Notes

- Uses PDO prepared statements for inserts/queries
- Password stored as hash (`password_hash`)
- `config.php` generated once by setup
- Setup lock file created at `data/install.lock`
- `data/.htaccess` blocks direct access on Apache
- No raw IP or IP hash stored

## Reinstall / Reset

To reinstall:

1. Delete `config.php`
2. Delete `data/install.lock`
3. (Optional) delete your SQLite DB file
4. Open `/setup.php` again

## Troubleshooting

- `500` on setup or collect:
  - Ensure PHP has SQLite enabled
  - Ensure DB directory is writable
- `403` on collect:
  - Check `allowed_origins` and `primary_site` values from setup
- No events showing:
  - Confirm tracker endpoint URL is correct
  - Check browser network tab for `collect.php` status codes
